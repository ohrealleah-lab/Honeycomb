# Code Review Skill
When reviewing code, systematically check for:
1. **Correctness**: Does the code actually do what it's supposed to?
2. **Edge cases**: Are null pointers, empty states, or timeout errors handled?
3. **Performance**: Are there obvious bottlenecks or unnecessary database queries?
4. **Security**: Are there any exposed secrets or injection risks?
